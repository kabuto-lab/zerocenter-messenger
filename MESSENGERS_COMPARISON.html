Where ME55 wins (or ties best-in-class)

  • Single static binary — 18 MB exe, no installer, no admin rights. Unique among the 11.
  • Anonymous sign-up — no phone, no email, no account. Only qTox / Keet match.
  • Wire-level obfuscation (ScrambleStream + IAT jitter + frame padding). Nobody else does this at the messenger layer; closest is Tor.
  • OS-keyring DEK + ChaCha20-Poly1305 at rest. iMessage has equivalent (Data Protection), nobody else has both.
  • Sealed sender + per-message Ed25519. Matches Signal; better than everyone else.
  • No subpoena target — no company, no centralised metadata. Only qTox / Keet match.
  • DHT mailbox without a server. Everyone else uses a server for offline delivery; qTox / Keet have weak offline support.
  • Group-chat key rotation on member leave matches Signal/WhatsApp; works without any central authority.

Where ME55 falls behind

  • Voice / video calls — totally missing. WA/Signal/everyone have them.
  • Mobile clients (iOS / Android) — not implemented; this kills mass-adoption.
  • Push notifications — depends on mobile, so also missing.
  • External security audit — only an in-house self-audit so far.
  • UDP / QUIC transport — currently TCP-only; weakens NAT-traversal vs qTox / Keet.
  • Public bootstrap network — user must arrange their own VPS; qTox has 50+ community nodes baked in.
  • File / image / voice messages — text only. Major UX gap.
  • Spam / abuse resistance — no rate limits, no reputation; ToxID-style spam attacks would land easily.
  • Post-quantum ratchet — Signal (PQXDH) and iMessage (PQ3) shipped this in 2023-2024.
  • Deniability — per-message Ed25519 sig is cryptographic proof of authorship (opposite of Signal's deniability).
⚠ Data accuracy notice. Mainstream-messenger behaviour evolves quickly (WhatsApp added E2E backup in 2021, Messenger went E2E-default in 2024, Signal shipped PQXDH in 2023, iMessage shipped PQ3 in 2024, Signal added usernames in 2024, etc.). Cells here reflect best knowledge as of early 2026; for specific recent versions please cross-check with the vendor's own security whitepaper.
