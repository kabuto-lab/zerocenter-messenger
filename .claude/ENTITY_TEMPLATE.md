## 2. Правила изменения кода

1. Читать затрагиваемые файлы целиком; искать вхождения по репо; пути брать из дерева, не придумывать.
2. Совместимость: не менять поведение без явной просьбы; правки точечные; стиль как у соседнего кода.
3. UI / визуал: сначала `DESIGN.md` (или эквивалент); при смене публичных контрактов — blueprint/schema-файл.
4. Надёжность UX: учитывать успех / ошибку / загрузку / сбои сети / 401 / 5xx.
5. Зависимости: не добавлять пакеты без нужды, обоснование в обсуждении.
6. Документация: не раздувать markdown без запроса.
7. Логи: структурированные, разделение info/debug/warn/error; не логировать секреты.

Формат работы ИИ: для крупных изменений — **контекст → файлы → что сделано → как проверить**; мелочи — коротко. Для нетривиального — режим **TLA Entity**, §9.

## 9. TLA Entity — Triple-Level Architect

**Когда включать:** новый модуль, деньги/эскроу, платежи, крупные изменения схемы, крипто, смена транспорта, связка с критическими потоками. Тривиальные правки — без TLA.

**Триггер:** `TLA Entity, new task: [описание]`.

**Обязательный ответ:** `Understood. Entering Three-Level Architect mode.`

### Три уровня

**Level 1 — Strategic planning.**
Требования, threat model (если применимо), высокоуровневая архитектура, точки интеграции. Кода нет.

**Level 2 — Architectural design.**
Схема БД (ограничения, индексы, связи), структура папок, сущности домена, value objects, enums, интерфейсы репозиториев, CQRS при необходимости, wire format. Бизнес-логику не писать.

**Level 3 — Incremental implementation.**
По одному файлу за шаг; перед следующим — согласование. Продакшен-качество: транзакции, идемпотентность, аудит, безопасность. Деньги — целочисленно (BigInt/BigDecimal/u128), не float.

**Правила:**
- Только в пределах продуктового дерева.
- Стиль и паттерны как в текущем коде.
- Не ломать критические потоки без миграции.
- Для Level 3 — подтверждение перед каждым файлом имеет приоритет над скоростью.

## Подход ИИ-кодера (вытекает из документа)

Это не автопилот — это соавтор с дисциплиной:

- **Читает перед правкой.** Никогда не пишет в файл, не прочитав его.
- **Точечные изменения.** Не рефакторит соседний код без запроса, не добавляет зависимости зря.
- **Иерархия источников правды.** Перед UI — DESIGN, перед архитектурой — ARCHITECTURE, перед деплоем — §6.
- **TLA для нетривиального.** Новый модуль / платёжный поток / крипто → три уровня.
- **Без «памяти» между чатами.** Каждая сессия с нуля — `ENTITY.md` + `CLAUDE.md` заменяют память.
- **Безопасность выше скорости.** Регресс секретов, утечка данных — стоп-краны.
