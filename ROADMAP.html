Phase 3 — End-to-end encryption Done (7 / 7 commits)

Hand-rolled Double Ratchet on primitives already in the dep tree (x25519-dalek, chacha20poly1305, hkdf, sha2, hmac, zeroize, subtle). No new crates compiled. Full plan in C:\Users\a3\.claude\plans\phase3-e2ee.md.
  • ① Identity prekey done
    X25519 signed prekey added to Identity; lazy migration of old identity.json; tests cover roundtrip, migration, tamper-and-regenerate. x25519-dalek "static_secrets" feature enabled.
  • ② Prekey exchange protocol done
    New libp2p request-response /zerocenter/prekey/1.0.0; prekeys_seen SQLite table with save_prekey / load_prekey; NodeCommand::FetchPrekey dispatches the request; inbound responses are signature-verified against the responder's PeerId-embedded Ed25519 key before caching.
  • ③ Ratchet + X3DH-lite modules done
    src/crypto/x3dh.rs (two-DH initial agreement) + src/crypto/ratchet.rs (full Double Ratchet per Signal spec: KDF_RK / KDF_CK, DH ratchet step, skipped-key cache with MAX_SKIP=1000, ChaCha20-Poly1305 AEAD with header bound into AAD, JSON state serialization). 13 unit tests cover X3DH agreement + ratchet roundtrip + out-of-order within / across ratchet step + tamper / wrong-AD / cross-session isolation / skipped-cap / JSON roundtrip.
  • ④ Wire-format flip done — irreversible change landed
    EncryptedPayload (dh + pn + n + ct + optional x3dh_eph) added to protocol/message.rs; protocol bumped to /zerocenter/direct-message/2.0.0; node.rs rewritten with pending-sends / pending-recvs queues that fire prekey-fetch on demand and drain on response. Send: existing session → encrypt; cached prekey → X3DH-initiator + first-message x3dh_eph; neither → queue+fetch. Recv: existing session → decrypt; payload has x3dh_eph + cached prekey → X3DH-responder bootstrap + decrypt; neither → queue+fetch. AEAD AD = length-prefixed sender_pid || recipient_pid.
  • ⑤ Session persistence done — bundled with ④
    ratchet_sessions SQLite table; save_session/load_session; persist_session() fires after every encrypt and decrypt; restore_session_if_persisted() lazily loads on first lookup. Plaintext-at-rest caveat documented in code; encryption-at-rest deferred to Phase 3.5.
  • ⑥ Local plaintext history deferred — folded into ④
    Decided to keep storing plaintext in existing messages.ciphertext column (it always was meant as "local view of conversation"). A separate messages_plaintext_local table is unnecessary churn for now; will revisit when commit ⑦ updates docs.
  • ⑦ README + threat-model section done
    README rewritten: Phase 3 Complete section documents Double Ratchet + X3DH-lite + prekey protocol + session persistence. New Threat Model section explicitly lists what an on-wire attacker cannot do, what they still can (initial-DM MITM, no async first message, no deniability, traffic analysis, DHT metadata), and what local-file access exposes (history / identity key / ratchet state — all plaintext at rest until Phase 3.5). Adds explicit "gap to Signal-equivalent" list and a flag that this codebase has not been audited.
Open decisions I'm making unless redirected: zero-nonce per message key · MAX_SKIP=1000 · serde_json for state blobs · session-state encryption deferred to Phase 3.5 · protocol version bumps to 2.0.0.
